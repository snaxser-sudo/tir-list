# Tier List Web App

Статическое web-only приложение для создания, редактирования и публикации tier list.

## Запуск

Откройте `index.html` в браузере или запустите локальный статический сервер:

```bash
python3 -m http.server 4173
```

Затем откройте `http://localhost:4173/`.

## Как поделиться с другом

Локальный адрес `localhost` или `127.0.0.1` работает только на вашем компьютере. Чтобы друг открыл приложение, разместите эти файлы на любом статическом хостинге:

- GitHub Pages
- Cloudflare Pages
- Netlify

Рабочий сценарий без backend:

1. Вы публикуете проект и получаете публичный адрес, например `https://username.github.io/tier-list/`.
2. Отправляете этот адрес другу.
3. Друг открывает сайт, составляет tier list и нажимает `Поделиться`.
4. Приложение создаёт ссылку вида `https://username.github.io/tier-list/#data=...`.
5. Друг отправляет вам эту ссылку, и вы сразу видите готовый результат.

Важно: без backend или внешнего хранилища один и тот же базовый адрес не может сам обновиться у другого человека после действий друга. Результат передаётся через новую ссылку с `#data=...`.

## Индексация

Проект закрыт от индексации через `robots.txt`, meta-теги `noindex` в `index.html` и заголовок `X-Robots-Tag` в `_headers` для хостингов, которые поддерживают этот файл.

## Как хранится состояние

- Черновик сохраняется в `localStorage` под ключом `tier-list-draft-v1`.
- Публичная ссылка хранит состояние прямо в hash URL: `#data=...`.
- Данные URL имеют приоритет над черновиком.
- Backend, база данных, регистрация и серверное сохранение не используются.

## WIP-лимиты

- `S`: максимум 1 карточка.
- `A`: максимум 3 карточки.
- `B`, `C`, `D`: без лимита.

Все изменения проходят через централизованную проверку состояния в `commitState`, включая drag-and-drop, импорт из URL и восстановление черновика.
